**Financial Transaction Data Engineering Platform**

A Python-based financial data engineering project designed to process, transform, and store transaction data for analytical and machine learning workflows. The platform automates data validation, customer entity generation, feature engineering, and ETL processes, loading structured transaction data into a normalized MySQL warehouse.

**_Overview_**

This project was built to simulate a production-style financial data pipeline capable of handling large transaction datasets. The system converts raw transaction records into structured relational datasets suitable for downstream analytics, reporting, and fraud-risk modeling.

_**Key Features**_
Automated processing of 6,600+ financial transaction records
Generation of 2,000+ unique customer entities
Data validation and normalization workflows using Pandas
Transaction-level feature engineering
Automated ETL pipelines using SQLAlchemy
Normalized MySQL warehouse design
Reusable analytics-ready datasets
Foundation for fraud detection and risk analytics applications
